## Bitcoin Custody â€” dVaults

At the core of Surge are **dVaults**: Taproot outputs encoded with multiple spend paths using **MAST (Merklized Alternative Script Trees)**. Only the executed branch is revealed, preserving privacy and efficiency.

### Spend Conditions
- **Repayment** â€” borrower + signer quorum co-sign repayment to unlock BTC.
- **Liquidation** â€” signer quorum executes liquidation if collateral ratio < threshold.
- **Migration** â€” reassigns signer set under a new epoch.
- **Unilateral Exit** â€” borrower can reclaim BTC after `OP_CHECKLOCKTIMEVERIFY` expiry.

### Security Properties
- **Bitcoin-native** â€” collateral never bridges; BTC never leaves Bitcoin.
- **Non-custodial** â€” no individual signer controls funds.
- **Threshold Signatures** â€” Lin22 Schnorr TSS (via [cb-mpc](https://github.com/coinbase/cb-mpc)) resists collusion or aborts.
- **Oracle-driven** â€” BTC/USD feed governs collateral ratios and liquidation triggers.

## Execution Environment

External chains **do not custody BTC** â€” they only mirror vault conditions and enforce protocol logic.

- **Event Mirroring** â€” deposits, debt issuance, repayments, and liquidations map 1:1 to Bitcoin spends.
- **Signer Epochs** â€” track signer rotations, validate signatures against the active set.
- **Collateral Ratios** â€” monitor vault health via oracles; liquidation events reconcile against Bitcoin.
- **Stability Pool** â€” depositors absorb undercollateralized debt and receive discounted BTC.

Execution runs on existing EVM-compatible chains with deterministic consensus (`reth` + CometBFT) for consistent state replication.

## Cross-Chain Liquidity

Stablecoin supply moves across ecosystems while BTC remains locked:

- **Circle CCTP v2** â€” routes liquidity across Ethereum, Arbitrum, Optimism, and more.
- **Canonical Events** â€” standardized signals (`AttestedDeposit`, `DebtIssued`, `DebtRepaid`, `LiquidationTriggered`) form the audit log.
- **Deterministic Reconciliation** â€” ensures stablecoin supply always matches BTC collateral locked in dVaults.


## End-to-End Model
- BTC is the **source of truth**, locked in Taproot contracts.
- Execution chains act as **state mirrors**, coordinating signer rules and debt balances.
- Users retain **sovereignty**: cooperative signer flows or unilateral exits guarantee recoverability.

This architecture enables **sovereign Bitcoin capital markets** â€” BTC secured by Taproot contracts, with programmable stablecoin liquidity and settlement across multiple chains.
