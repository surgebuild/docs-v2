# Submit Proofs to Bitcoin

## Create a Surge Wallet in Keplr and request from faucet

### Create a Surge Wallet in Keplr

refer [Connect Keplr Wallet](/developer-guides/wallets-tokens/connect-keplr)

### Request from faucet

refer [Request from faucet](/developer-guides/wallets-tokens/get-tokens)

### Intialize Surge SDK

```bash
npm install @surgebuild/surge-sdk
```

After installing the Surge SDK, you can initialize it with the following code:

```typescript
import { SurgeClient } from "@surgebuild/surge-sdk";

const client = new SurgeClient();

await client.init(
  "a618676f8d7ed0322b98ca53f580b9c5c9d87d8a9ec1b60fe007b2b1000a2cbd" //add your private key from Keplr Wallet
);
```

### Submit proof parameters

```typescript
{
  "creator": "string",
  "methodId": "string",
  "proof": "string",
  "proofSystem": "string",
  "publicInputData": "string"
}
```

### Submit Proofs

```typescript
const res = await client.submitProof(
  "srg15428vq2uzwhm3taey9sr9x5vm6tk78ewcemlpx", //creator public address
  "./method.bin", //methodId file path
  "./state_verifier.proof", //proof file path
  "RISCZERO", //proof system type "RISCZERO" or "SP1"
  "./receipt.pub" //public input data file path
);
console.log(res);
```

Proof file will differ based on the proof system you are using.

For RISCZERO, you need to pass the `receipt.pub` file as a public input data, whereas for SP1, you can pass the public input data as an empty string.
