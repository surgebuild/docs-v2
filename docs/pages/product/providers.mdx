## Decentralized Network Nodes
Infrastructure providers are the backbone of Surge‚Äôs decentralized architecture. 
They enable trust-minimized operations for custody and execution. 
There are two key roles:

- Signer Nodes (for dVault custody and Taproot logic enforcement)
- Execution Nodes (for Bitcoin-aware loan lifecycle management)

### üîë Signer Network

Surge establishes a distributed and decentralized custody framework using multi-party computation (MPC) and threshold signature schemes (TSS). 
The signer network's core responsibilities include:

- Running lightweight clients that co-sign Bitcoin transactions alongside users for dVaults
- Executing Taproot-based scripts according to predetermined loan logic and conditions

Initially, the network will operate as a permissioned set of up to 20 signers, with capacity to scale to 50 nodes. 
Participation requires:

- Running minimal-cost infrastructure
- Staking BTC to ensure economic alignment

Eventually, this network will evolve into a fully permissionless signer set, expanding decentralization and resilience.
